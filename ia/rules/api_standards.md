# Стандарты API

## Общие принципы
- Использовать RESTful-подход
- Версионирование через префикс `/api/v1/`
- Все ответы в формате JSON
- Единая структура ответов
- Использовать HTTP-статусы в соответствии с семантикой

## Структура ответов

Успешный ответ:
```json
{
  "success": true,
  "data": { ... },
  "message": "Описание операции"
}
```

Ответ с ошибкой:
```json
{
  "success": false,
  "errors": [
    {
      "field": "email",
      "message": "Поле email обязательно для заполнения."
    }
  ],
  "message": "Валидация не пройдена"
}
```

## Методы и эндпоинты

| Метод | Эндпоинт | Описание |
|-------|---------|----------|
| GET | `/api/v1/resource` | Получение списка |
| GET | `/api/v1/resource/{id}` | Получение одного элемента |
| POST | `/api/v1/resource` | Создание |
| PUT | `/api/v1/resource/{id}` | Полное обновление |
| PATCH | `/api/v1/resource/{id}` | Частичное обновление |
| DELETE | `/api/v1/resource/{id}` | Удаление |

## Аутентификация
- Использовать Laravel Sanctum
- Токен передаётся в заголовке `Authorization: Bearer <token>`
- Защита маршрутов через middleware `auth:sanctum`

## Валидация
- Все входные данные должны быть валидированы
- Использовать FormRequest для сложной валидации
- Возвращать 422 Unprocessable Entity при ошибках валидации

## Ограничение скорости
- Использовать middleware `throttle` для защиты от перегрузки
- Настроить лимиты в зависимости от роли пользователя

## Фильтрация, сортировка, пагинация
- Фильтрация: `/api/v1/reports?status=draft&customer_id=5`
- Сортировка: `/api/v1/reports?sort=-created_at` (минус для убывания)
- Пагинация: включена по умолчанию, стандартный размер страницы — 15

## Поиск
- Поиск по нескольким полям: `/api/v1/employees?search=Иванов`
- Использовать Laravel Scout при необходимости полнотекстового поиска

## Ошибки
- Использовать стандартные HTTP-статусы:
  - 401 Unauthorized — отсутствие аутентификации
  - 403 Forbidden — недостаточно прав
  - 404 Not Found — ресурс не найден
  - 422 Unprocessable Entity — ошибка валидации
  - 500 Internal Server Error — внутренняя ошибка сервера
