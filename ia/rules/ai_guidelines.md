# Руководство для взаимодействия с ИИ

## Общие принципы
- Использовать ИИ для ускорения рутинных задач
- Всегда проверять сгенерированный код перед коммитом
- Не передавать чувствительные данные (пароли, ключи, персональные данные)
- Использовать промты из папки `ia/prompts/`

## Этапы разработки

### 1. Анализ требований
- Использовать промт `analyze_requirements.md`
- Выделить основные сущности, связи, бизнес-правила
- Определить роли и права доступа

### 2. Проектирование
- Создать ER-диаграмму
- Определить структуру миграций
- Продумать API-интерфейсы
- Использовать промт `design_database.md`

### 3. Генерация кода
- Использовать специализированные промты для каждого типа файла
- Проверить соответствие стилю кодирования и архитектуре
- Настроить автозагрузку классов

### 4. Тестирование
- Генерировать тесты с помощью промта `write_test.md`
- Покрыть основные сценарии: позитивные, негативные, граничные
- Использовать фабрики для создания тестовых данных

### 5. Отладка
- Использовать промт `debug_error.md` для анализа ошибок
- Предоставлять полный стектрейс, код и контекст
- Проверять конфигурацию и окружение

### 6. Документирование
- Генерировать документацию по API
- Обновлять архитектурные диаграммы
- Вести changelog

## Ограничения
- ИИ не заменяет разработчика
- Критические решения принимает человек
- ИИ может допускать ошибки в коде и логике
- Проверка безопасности обязательна

## Рекомендации
- Регулярно обновлять промты
- Делиться успешными шаблонами
- Вести журнал использования ИИ
- Обучать команду работе с ИИ-ассистентами
