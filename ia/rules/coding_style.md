# Стиль кодирования

## PHP (PSR-12)
- Использовать 4 пробела для отступов
- Максимальная длина строки — 120 символов
- Имена классов в UpperCamelCase
- Имена методов и переменных в camelCase
- Константы — только заглавными буквами с подчёркиванием
- Использовать типизацию везде, где возможно
- Добавлять PHPDoc-комментарии для всех методов

## Blade-шаблоны
- Использовать короткие теги: `@if`, `@foreach`, `@isset`
- HTML-теги и атрибуты — строчные
- Использовать двойные кавычки для строк в атрибутах
- Классы Tailwind CSS группировать по логике (layout, typography, spacing и т.д.)

## JavaScript
- Использовать 2 пробела для отступов
- Имена переменных в camelCase
- Предпочитать `const` и `let` вместо `var`
- Использовать стрелочные функции, где уместно
- Минимизировать использование внешних библиотек, использовать Alpine.js для интерактивности

## Именование файлов
- Модели: `SingularUpperCamelCase.php` (например, `Report.php`)
- Контроллеры: `PluralUpperCamelCaseController.php` (например, `ReportsController.php`)
- Миграции: `YYYY_MM_DD_HHMMSS_description.php`
- Шаблоны: `snake-case-with-dashes.blade.php`

## Git
- Сообщения коммитов на английском языке
- Формат: `<тип>(<модуль>): <описание>`
  - Типы: feat, fix, docs, style, refactor, test, chore
  - Пример: `feat(reports): add PDF export functionality`
