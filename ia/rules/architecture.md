# Архитектурные принципы

## Общие принципы
- Следовать принципам SOLID
- Минимизировать логику в контроллерах (thin controllers)
- Выносить бизнес-логику в сервисы и репозитории
- Использовать DTO для передачи данных между слоями
- Применять CQRS для сложных операций (отдельные модели для чтения и записи)

## Модели
- Каждая модель соответствует одной таблице в БД
- Использовать Eloquent-отношения для связей
- Выносить сложные запросы в локальные или глобальные scope
- Использовать traits для повторяющегося поведения (SoftDeletes, HasFactory и др.)

## Контроллеры
- Использовать resource-контроллеры для CRUD-операций
- Валидация — через FormRequest
- Авторизация — через middleware и методы authorize
- Возвращать view для страниц и JSON для API

## Сервисы
- Каждый сервис отвечает за одну бизнес-сущность или процесс
- Сервисы получают данные через DTO
- Сервисы возвращают DTO или модели
- Сервисы не должны напрямую работать с запросом или сессией

## Репозитории
- Абстрагируют доступ к данным
- Реализуют паттерн Repository
- Могут использовать кэширование
- Предоставляют методы для сложных выборок

## DTO (Data Transfer Objects)
- Используются для передачи данных между слоями
- Создаются через `spatie/data-transfer-object`
- Имеют строгую типизацию
- Не содержат логики

## Livewire
- Каждый компонент — отдельный класс и шаблон
- Использовать `wire:model` для двусторонней привязки
- Выносить сложную логику в методы
- Использовать `wire:loading` для индикаторов загрузки
- Оптимизировать рендеринг с помощью `@entangle` и `defer`
