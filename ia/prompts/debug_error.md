# Промт: Отладка ошибки

## Контекст
Вы анализируете и устраняете ошибку в Laravel-приложении MarineService Manager.

## Инструкции
1. Получите полный стектрейс ошибки
2. Определите тип ошибки: синтаксическая, логическая, конфигурационная, связанная с БД и т.д.
3. Проверьте контекст: окружение, входные данные, состояние системы
4. Воспроизведите ошибку в локальном окружении
5. Используйте отладку: dd(), logger, xDebug
6. Предложите решение с объяснением причины
7. Добавьте тест для предотвращения регрессии

## Требования
- Не изменять код без понимания последствий
- Проверять влияние на другие модули
- Следовать принципам безопасности
- Обновлять документацию при необходимости

## Пример
```
### Описание проблемы
При создании отчёта возникает ошибка 500 Internal Server Error.

### Стекстрейс
```
PDOException: SQLSTATE[23000]: Integrity constraint violation: 1048 Column 'customer_id' cannot be null
```

### Контекст
- Окружение: local
- Версия Laravel: 12.0
- Входные данные: {"title": "Акт", "status": "draft"}
- Шаги для воспроизведения:
  1. Открыть форму создания отчёта
  2. Ввести название и статус
  3. Нажать "Создать"

### Диагностика
Ошибка возникает потому что поле customer_id не передаётся в запросе, но в миграции оно помечено как NOT NULL. Валидация в FormRequest не требует это поле.

### Решение
Добавить 'customer_id' => 'required|integer|exists:customers,id' в правила валидации StoreReportRequest.

### Проверка
1. Повторить шаги воспроизведения
2. Убедиться, что возвращается 422 с ошибкой валидации при отсутствии customer_id
3. Проверить, что при передаче валидного customer_id отчёт создаётся

### Профилактика
Всегда проверять соответствие правил валидации и схемы БД. Использовать тесты для проверки граничных случаев.
```
